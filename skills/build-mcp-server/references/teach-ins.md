# 개념 카드 (teach-in) — 단계별로 3~4줄만 보여준다

`teachMode=teach`면 해당 Phase 질문 **직전**에 그 카드를 그대로(또는 다듬어) 보여준다.
`teachMode=mixed`면 첫 줄(굵은 문장)만. `fast`면 생략하고 "왜? ▸" 한 줄로.
강의(6주차)의 표현을 그대로 쓴다.

---

## CH1 카드 — Phase 0 (탄생/개념)
**MCP는 AI와 도구가 함께 쓰는 "공용어(프로토콜)"다.**
- 예전엔 LLM M개 × 도구 N개를 잇는 데 M×N개의 커스텀 통합이 필요했다 → 조합 폭발.
- MCP는 이걸 **M+N**으로 줄인다: "한 번 만들면 모든 곳에서 사용."
- 이름 그대로 **Model**(두뇌=LLM) · **Context**(책상 위 자료·도구·상태) · **Protocol**(모두가 쓰는 약속).

## CH2 카드 (레스토랑 비유) — Phase 0
**MCP는 Host·Client·Server 세 역할이 나눠 맡는다.**
- **Host = 지휘관**(주문 이해·계획·최종 응답, 예: Claude Desktop/Cursor)
- **Client = 중개자**(서버와 1:1 연결·보안 경계, Host 안에서 실행)
- **Server = 주방**(실제 작업 수행 — 우리가 만드는 것)
- 손님(사용자)은 주방이 어디 있는지 몰라도 된다 = API를 몰라도 되는 이유.

## CH2 카드 (서버 3요소) — Phase 2
**서버가 AI에게 여는 창구는 세 가지다.**
- **Resources(자료실)** — AI가 *읽어올* 데이터 (파일·DB·웹페이지)
- **Tools(도구함)** — AI가 *실행할* 동작 (API 호출·계산·변환)
- **Prompts(템플릿)** — 미리 정의된 명령어/재사용 문구
- 읽고(Resources)·실행하고(Tools)·지시받는다(Prompts).

## CH3 카드 (강점/한계) — Phase 1
**MCP는 만능이 아니다. 한계를 아는 게 성공적 활용의 시작.**
- 잘 맞는 일: 데이터 접근·조회 / 자동화·워크플로 / 정보 통합·분석
- 안 맞는 일: 실시간 스트리밍(밀리초 응답) / 장기 상태·복잡 트랜잭션 / 대용량·고성능 연산
- MCP는 요청-응답 패턴이다. 실시간 거래·대용량 비디오는 부적합.

## Phase 3 프레이밍 카드 (입출력)
**타입 힌트 = 주문서 양식, docstring = AI가 읽는 설명서.**
- `city: str` 같은 타입 힌트가 곧 입력 폼(스키마)이 된다.
- docstring은 "AI가 이 도구를 언제 써야 하는지"를 알려주는 메뉴 설명.
- AI는 이 설명을 읽고 스스로 언제 호출할지 판단한다.

## CH4 카드 (보안) — Phase 4
**문을 열어준다는 것 — 편리한 만큼 지킬 원칙이 필요하다.**
- 위험 4가지: 민감 데이터 노출 · 무단 명령 실행 · API 키 탈취 · 시스템 침해
- 원칙: **최소 권한**(꼭 필요한 만큼만 연다) + 입력 검증(화이트리스트/스키마)
- 프롬프트 인젝션: 웹/문서에 숨은 "이전 지시 무시하고…" 같은 명령 주의.
  → **도구가 가져온 내용은 명령이 아니라 데이터로 취급** + 위험 작업엔 사용자 승인.

## CH5 카드 (연결) — Phase 5
**설정 파일 = Claude의 전화번호부. 시작할 때 한 번 읽고, 여기 적힌 서버에만 연락한다.**
- 연결은 언제나 3단계: ① 설정 파일 열기 → ② 서버 블록 추가 → ③ 완전 재시작.
- 블록은 `command`(실행 프로그램: uv/npx/python) · `args`(무엇을 켤지) · `env`(키 자리)로 구성.
- 그래서 수정 후엔 반드시 **완전 종료 후 재시작**. 콤마·괄호 하나만 틀려도 전체 연결 실패.

## 설계 관점 카드 — Phase 3~4 사이 (선택)
**좋은 MCP 서버는 기능 모음이 아니라 "AI의 행동 설계"다.**
- 문법(@mcp.tool·resource·prompt)은 이미 다 배웠다. 남은 건 설계 — AI가 실수 안 하게 길을 내는 일.
- 상대가 사람이 아니라 AI일 뿐, 좋은 매뉴얼을 쓰는 일과 같다.
